# Frontend Design Helper Skill

Use this skill when designing or polishing the frontend of a Webrivio client website.

The goal is to avoid generic AI-looking frontend work and create a distinctive, polished, production-grade interface.

## Core Design Standard

Before coding visual sections, choose a clear design direction.

Decide:
- Overall aesthetic
- Typography direction
- Color relationship
- Layout rhythm
- Image treatment
- Motion style
- Spacing system
- Visual hierarchy
- Section rhythm

Avoid:
- Generic SaaS layouts
- Predictable card grids
- Purple/blue gradients unless specifically justified
- Flat pages with no depth
- Random decorative blobs
- Overused Inter-only typography feel
- Text-heavy sections
- Repeating identical section layouts
- Motion that is only fade-in-on-scroll

Prefer:
- Strong first viewport composition
- Distinctive typography pairing
- Confident whitespace
- Intentional color palette
- Image-led layouts
- Strong contrast between sections
- Premium hover states
- Scroll-driven storytelling when useful
- Beautiful empty space
- Memorable hero structure
- Real or placeholder images treated as design objects

## Webrivio Design Translation

For Webrivio local-business websites, visual design must still support conversion.

Design choices should make the business feel:
- More trusted
- More premium
- Easier to contact
- More credible
- Less generic
- More memorable

Do not create art that hurts conversion.

Do not hide CTAs.

Do not make navigation confusing.

## Visual Rhythm Rule

Every 1-2 sections, change the rhythm.

Use combinations like:
- Split hero
- Full-bleed media
- Sticky panel
- Horizontal scroll gallery
- Large typography break
- Gallery mosaic
- Process timeline
- Visual CTA strip
- Interactive card stack
- Map/service-area visual
- Scroll-pinned story section

## Motion Rule

Motion should feel alive, not gimmicky.

Use:
- Framer Motion for microinteractions and reveals
- GSAP ScrollTrigger for meaningful scroll-driven sections
- Hover states that feel premium
- Smooth staggered reveals
- Image movement tied to scroll where useful

Avoid:
- Bouncy animations
- Slow animations
- Over-animation
- Scroll hijacking
- Motion that competes with CTAs

## Final Design Audit

Before finishing a frontend design pass, check:
- Does the first viewport feel expensive?
- Is the site visually led, not text led?
- Is typography intentional?
- Do sections have rhythm?
- Are images treated beautifully?
- Is there at least one memorable visual moment?
- Are CTAs still clear?
- Does mobile feel polished?
- Does it avoid generic AI aesthetics?

If not, revise the design directly.
